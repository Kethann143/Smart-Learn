# Smart Learn Architecture

Modular, scalable codebase structure designed for cross-platform execution (Web, Electron Desktop, Capacitor Mobile).

## Project Folder Directory Purpose

- **`frontend/`**: The Single Page Application UI view templates, core controllers, styling themes, and offline data orchestration helper utilities.
  - `screens/`: Controller logic orchestrator (`app.js`).
  - `layouts/`: Core HTML skeleton templates (`index.html`).
  - `themes/`: CSS visual design frameworks (`styles.css`).
  - `utils/`: Offline SQLite bridging, auth sync state tracking, AI tutors, and service worker caches.
  - `assets/`: App logo media assets and PWA `manifest.json`.
- **`backend/`**: Express API server backend routing. Contains `server.js` and directories for modular APIs, security middlewares, controllers, and models.
- **`database/`**: Persistence databases layer. Houses the offline SQLite file (`smart_learn.db`) and Firebase Cloud Firestore collection layout schemas/security rules.
- **`app/`**: Electron application desktop main shell entry point and secure IPC preloads.
- **`android/`**: Mobile container build configurations, native assets, and permissions declarations generated by Capacitor.
- **`web/`**: Reserved components, assets, public folders, and page templates for deployment as a standard static webapp.
- **`tests/`**: Automated E2E WebDriver tests, Mobile Appium setups, and load-telemetry simulations.
- **`scripts/`**: Development automation build compilers and database checking scripts.
- **`docs/`**: Design documentations, vulnerability reports, and telemetry spreadsheets.
